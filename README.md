## рҹҢҹ Overview

**YUR FRANKLIN OS** (formerly Sovereign Genesis Platform v2.0) is a sophisticated, VS Code-style IDE and complete software factory designed to build, certify, and deploy enterprise-grade applications using AI-powered automation. The platform combines cutting-edge AI agents, self-healing code generation, and comprehensive quality assurance pipelines to deliver production-ready software.

## рҹҡҖ Getting Started

### Prerequisites

- **Python**: 3.11 or higher
- **Node.js**: 18.x or higher
- **MongoDB**: 4.4 or higher (or MongoDB Atlas)
- **Git**: Latest version
- **XAI API Key**: Required for Grok agent

### Installation

#### 1. Clone the Repository

```bash
git clone https://github.com/YUR-AI-CREATIONS/YUR-FRANKLIN-OS.git
cd YUR-FRANKLIN-OS
```

#### 2. Backend Setup

```bash
cd backend

# Create virtual environment
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate

# Install dependencies
pip install -r requirements.txt

# Configure environment variables
cat > .env << EOF
MONGO_URL=mongodb://localhost:27017
DB_NAME=franklin_os
CORS_ORIGINS=*
XAI_API_KEY=your_xai_api_key_here
EMERGENT_LLM_KEY=your_emergent_llm_key_here
EOF

# Run the backend server
uvicorn server:app --reload --host 0.0.0.0 --port 8000
```

#### 3. Frontend Setup

```bash
cd ../frontend

# Install dependencies
npm install
# or
yarn install

# Configure environment variables
cat > .env << EOF
REACT_APP_BACKEND_URL=http://localhost:8000
WDS_SOCKET_PORT=443
EOF

# Start the development server
npm start
# or
yarn start
```

#### 4. Access the Application

- **Frontend**: http://localhost:3000
- **Backend API**: http://localhost:8000
- **API Documentation**: http://localhost:8000/docs

### Optional: Local LLM Setup (Ollama)

For cost-free local inference:

```bash
# Install Ollama
curl https://ollama.ai/install.sh | sh

# Pull recommended models
ollama pull llama3.1:8b
ollama pull codellama:13b
ollama pull phi3:14b

# Configure hybrid mode in the app
# LLM Selector вҶ’ Choose "Hybrid" or "Local"
```

## рҹ“ҡ Usage

### Basic Workflow

#### 1. **Enter Requirements**

Navigate to the IDE and enter your project requirements in natural language:

```
"Build a task management app with user authentication, 
real-time updates, and PostgreSQL database"
```

#### 2. **Answer Clarifications**

The Socratic Engine will ask clarifying questions across categories:
- Authentication (OAuth? JWT? Magic links?)
- Data persistence (Relations? Migrations?)
- Scalability (Expected users? Geographic distribution?)
- Security (Compliance requirements? Encryption?)

Continue until confidence reaches 99.5%+

#### 3. **Select Technology Stack**

Choose from 40+ pre-configured technologies or let the AI recommend:
- Frontend: Next.js + React + Tailwind
- Backend: FastAPI + PostgreSQL
- Auth: Clerk or Auth0
- Hosting: Vercel (frontend) + Railway (backend)

#### 4. **Generate Code**

The Genesis Pipeline executes through 8 stages:
1. **Inception**: Requirement validation
2. **Specification**: Detailed spec generation
3. **Architecture**: System design
4. **Construction**: Code generation
5. **Validation**: Testing
6. **Evolution**: Optimization
7. **Deployment**: Deployment prep
8. **Governance**: Compliance check

#### 5. **Review & Deploy**

- Review generated code in `/generated/[ProjectName]/`
- Run quality gates and security scans
- Deploy with one click to Vercel, Railway, or AWS

### Command Examples

```bash
# Socratic analysis
POST /api/analyze
{
  "prompt": "Build an e-commerce platform"
}

# Initialize Genesis project
POST /api/genesis/project/init
{
  "name": "EcommerceApp",
  "description": "Full-featured e-commerce platform"
}

# Generate code artifacts
POST /api/build/generate
{
  "project_id": "proj_12345",
  "tech_stack": {
    "frontend": "nextjs",
    "backend": "fastapi",
    "database": "postgresql"
  }
}

# Write to disk
POST /api/build/write
{
  "project_id": "proj_12345"
}
```
